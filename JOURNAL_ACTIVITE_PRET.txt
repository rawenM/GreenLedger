ÉTAPE 1: Créer la table MySQL (5 min)
────────────────────────────────────────────────────────────────
Méthode 1 - Ligne de commande:
   mysql -u root -p greenledger < database_audit_log.sql

Méthode 2 - MySQL Workbench:
   1. Ouvrir MySQL Workbench
   2. Ouvrir database_audit_log.sql
   3. Exécuter le script (Ctrl+Shift+Enter)

Vérification:
   SELECT COUNT(*) FROM audit_log;
   → Vous devriez voir 5 logs de test


ÉTAPE 2: Intégrer dans les contrôleurs (10 min)
────────────────────────────────────────────────────────────────
Ajoutez ces lignes dans vos contrôleurs:

📝 LoginController.java
   // Après connexion réussie (ligne ~150)
   AuditLogService.getInstance().logLogin(user, "127.0.0.1");
   
   // Après connexion échouée (ligne ~160)
   AuditLogService.getInstance().logLoginFailed(
       emailField.getText(), 
       "Mot de passe incorrect", 
       "127.0.0.1"
   );

📝 AdminUsersController.java
   // Dans handleValidateUser() après succès
   AuditLogService.getInstance().logAdminValidateUser(currentUser, user);
   
   // Dans handleBlockUser() après succès
   AuditLogService.getInstance().logAdminBlockUser(currentUser, user);
   
   // Dans handleDeleteUser() après succès
   AuditLogService.getInstance().logAdminDeleteUser(currentUser, user);
   
   // Dans showFraudDetails() au début
   AuditLogService.getInstance().logAdminViewFraud(currentUser, user);


ÉTAPE 3: Tester (5 min)
────────────────────────────────────────────────────────────────
1. Lancer l'application: run.bat
2. Se connecter
3. Vérifier dans MySQL:
   SELECT * FROM audit_log ORDER BY created_at DESC LIMIT 10;


┌───────────────────────────────────────────────────────────────┐
│  ⚠️  IMPORTANT: ENREGISTREMENT AUTOMATIQUE                    │
└───────────────────────────────────────────────────────────────┘

L'enregistrement est AUTOMATIQUE et NE PEUT PAS être désactivé.

✅ L'admin PEUT:
   • Consulter les logs
   • Filtrer par date, type, utilisateur
   • Rechercher dans les logs
   • Exporter (si implémenté)

❌ L'admin NE PEUT PAS:
   • Désactiver l'enregistrement
   • Supprimer des logs
   • Modifier des logs

Pourquoi? C'est essentiel pour:
   • Conformité RGPD
   • Sécurité et traçabilité
   • Audit légal
   • Détection d'activités suspectes

Avantages:

plusieurs tentatives de connexion échouées ou des actions admin
