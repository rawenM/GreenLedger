🎯 RÉPONSES AUX QUESTIONS FRÉQUENTES
═══════════════════════════════════════════════════════════════

Q: "Pourquoi Gmail API et pas SMTP?"
───────────────────────────────────────────────────────────────
"Gmail API utilise OAuth2, ce qui est beaucoup plus sécurisé que 
SMTP qui nécessite un mot de passe en clair dans le code. De plus, 
Gmail API offre de meilleures garanties de délivrabilité et est 
utilisé par les applications professionnelles. C'est la méthode 
recommandée par Google."


Q: "Comment fonctionne exactement la détection de fraude?"
───────────────────────────────────────────────────────────────
"Le système analyse 10 indicateurs en temps réel lors de chaque 
inscription. Chaque indicateur a un poids spécifique. Par exemple:
- Email jetable: +30 points
- VPN détecté: +25 points
- Pattern suspect: +20 points

Le score final détermine le niveau de risque. Si le score dépasse 
75, le système recommande de bloquer. Entre 50 et 75, une vérification 
manuelle. En dessous de 50, l'utilisateur est considéré comme sûr. 
Tout ça est automatique et ne nécessite aucune intervention manuelle."


Q: "Pourquoi Chart.js?"
───────────────────────────────────────────────────────────────
"Chart.js est léger, gratuit, open-source, et offre des graphiques 
interactifs professionnels. Il est utilisé par des millions de sites 
web et s'intègre facilement avec JavaFX via WebView. Les alternatives 
comme D3.js sont plus complexes, et les bibliothèques Java natives 
sont moins modernes visuellement."


Q: "Le code à 6 chiffres est-il vraiment sécurisé?"
───────────────────────────────────────────────────────────────
"Oui, pour plusieurs raisons:
1. Généré avec SecureRandom qui est cryptographiquement sûr
2. Expire après 10 minutes seulement
3. Stocké uniquement en mémoire, jamais en base de données
4. Un seul essai possible avant expiration
5. C'est le même système utilisé par Google, Facebook, et WhatsApp

Avec 1 million de combinaisons possibles (000000 à 999999) et 
10 minutes d'expiration, c'est statistiquement très sécurisé."


Q: "Combien de temps pour développer tout ça?"
───────────────────────────────────────────────────────────────
"[Adaptez selon votre situation]
J'ai travaillé sur ce projet pendant [X semaines/mois]. La partie 
la plus complexe était l'intégration de la détection de fraude et

Q: "Quelles sont les technologies utilisées?"
───────────────────────────────────────────────────────────────
"Frontend: JavaFX avec FXML pour l'interface
Backend: Java 17 avec architecture MVC
Base de données: MySQL avec JDBC
APIs: Gmail API, Google reCAPTCHA, Chart.js
Sécurité: Bcrypt pour les mots de passe, OAuth2 pour Gmail
Outils: Maven pour la gestion des dépendances"


Q: "Pourquoi JavaFX et pas une application web?"
───────────────────────────────────────────────────────────────
"JavaFX offre plusieurs avantages pour ce projet:
1. Performance native (pas de navigateur requis)
2. Intégration facile avec MySQL
3. Sécurité: pas d'exposition directe sur internet
4. WebView permet d'intégrer Chart.js et reCAPTCHA
5. Expérience utilisateur fluide et réactive

Pour une plateforme financière, la sécurité d'une application 
desktop est un avantage."
